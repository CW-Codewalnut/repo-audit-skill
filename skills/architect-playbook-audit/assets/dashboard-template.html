        Ben's Architect Playbook

        Use the source playbook when one audit area needs a deeper, standalone pass.

        This dashboard consolidates the relevant Architect Playbook audits into one leadership-friendly report. For deeper work on a specific area, run the original Ben's Architect Playbook skill directly, for example security, testing, TypeScript, accessibility, or architecture.

        The original playbook writes full per-audit artifacts under .architect-audits/<audit-name>/, including layered findings, a Layer 0 snapshot, machine-readable JSON, and optional implementation plans.

        Open CW-Codewalnut/ArchitectPlaybook
        1. Clone the playbook repository and open it in Claude Code.
        2. Install the slash commands with /install-architect-playbook-globally, or use /install-architect-playbook-locally inside a target project.
        3. Prepare the target project with /pre-audit-setup so graph-aware audits can use the repository map.
        4. Run a focused audit such as /security-audit --learn, /testing-audit --with-run, or /architecture-audit --worktree.
        5. Use --worktree for isolated parallel audits, and use audit-specific --with-* flags when you want optional runtime, network, or existing report enrichment.
